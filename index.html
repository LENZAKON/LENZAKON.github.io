Promise

Promise实例的状态变为Resolved,就会触发then方法绑定的回调函数。

Promise创建后会立即执行，而then绑定的回调函数会在当前脚本的所有同步任务执行完后再执行

Promise封装Ajax

在一个Promise实例中的resolve方法中使用另一个Promise实例作为参数，及一个异步操作的结果是返回另一个异步操作。

两个Promise实例是同时进行的，当p2在1秒后状态改变时，由于返回的是另一个Promise，所以把后面的then和catch控制权交给p1，由p1在2秒后决定

Peomose.prototype.then

then返回的是一个新的Promise实例，所以在then后面可以再调用then。
依次指定多个回调函数，第一个回调函数执行的结构(return值)，作为参数传入第二个回调函数，让回调函数顺序执行

当前一个回调函数返回的还是Promise对象时，后一个回调函数要等待该Promise对象的状态发生变化，才被调用。

第一个then返回的是另一个Promise对象，第二个then指定的回调函数要等新的Promise对象状态发生变化。如果为resolve，调用funcA，如果为reject，调用funcB。

Promise.prototype.catch

当异步操作抛出错误时，状态变为reject，调用catch方法指定的回调函数，then方法中指定的回调抛出错误时也会被catch捕获。

所以在第一个then中指定第二个参数时，catch则不会执行。
reject方法的作用,等同于抛出错误。建议总是使用catch方法,而不使用then方法的第二个参数

如果Promise状态变为resolve，再抛出错误是无效的。

Promise错误是具有“冒泡”性质的，一直向后传递，直到被捕获为止。

catch会捕获三个Promise对象，一个由getJSON产生,两个由then产生。

如果没有使用catch指定错误处理的回调函数，Promise对象的错误不会传递到外层代码，即不会有反应（没有任何输出）

没有任何输出

Promise指定在下一轮“事件循环”中再抛出错误，冒泡到外层成了未捕获错误，此时，Promise函数体已经运行结束，所以错误是在Promise函数体外抛出的

Node 中监听未被捕获的reject错误

第一个是错误对象,第二个是报错的 Promise 实例

catch返回的也是Promise对象，后面可以接着调用then。如果一个异步操作有错误，则会执行catch，再执行后面的then，如果没有，则跳过catch执行then

catch中还可以再抛出错误，如果后面还有catch，则会被捕获，否则，则不会被捕获，也不会传递到外层。

  • 当p1,p2,p3都为fulfilled，p才为fulfilled。p1,p2,p3返回值组成数组，传给p的回调函数
  • p1,p2,p3只要有一个为rejected，p为rejected。第一个被rejected的实例的返回值，会传递给p的回调函数

  • 参数中只要有一个改变，p的状态就改变。率先改变的实例的返回值就会被传递给p的回调函数

Promise.resolve()

将现有对象转化成Promise对象

  • 参数是一个Promise实例。不做任何修改，直接返回
  • 参数是一个thenable对象。先将这个对象转化成Promise对象，然后立刻执行thenable的then方法。
  • 参数不是具有then方法的对象，则Promise.resolve方法返回一个新的Promise对象，状态为resolve。
  • 不带任何参数，直接返回一个resolved状态的Promise对象。

立即resolve的Promise对象，是在本轮“事件循环”的结束时，而不是在下一轮“事件循环”的开始时

Iterator和for...of循环

发表于 2018-01-19

Iterator

  • 概念：
    是一种接口，任何数据结构（set, map, arry, Object）只要部署Iterator接口，就可以完成遍历。

  • 作用：

    • 为各种数据结构提供统一，简便的访问接口。
    • 使数据结构的成员能够按某种次序排列。
    • ES6 创造了一种新的遍历命令for…of 循环, Iterator 接口主要供for…of 消费

  • 遍历过程：

    • 创建一个指针指向数据结构的起始位置。Iterator对象本质就是一个指针对象
    • 使用指针对象的next方法，指向第一个成员
    • 通过next方法直到数据结构的结束位置。next返回的是有value（当前成员值）和done（遍历是否结束）两个属性的对象。

数据结构的默认Iterator接口

  • 默认部署

    原生具备Iterator接口非数据结构：数组，某些类似数组的对象，Set，Map。其他数据结构（主要为对象）的Iterator接口，都需要自己在Symbol.iterator属性上部署，这样才会被for...of循环遍历。严格地说,对象部署遍历器接口并不是很必要,因为这时对象实际上被当作 Map 结构使用, ES5 没有 Map 结构,而 ES6 原生提供了。

默认的 Iterator 接口部署在数据结构的Symbol.iterator 属性,或者说,一个数据结构只要具有Symbol.iterator 属性,就可以认为是 “ 可遍历的 ” ( iterable )。
调用Symbol.iterator 方法,就会得到当前数据结构默认的遍历器生成函数。Symbol.iterator 本身是一个表达式,返回 Symbol 对象的iterator 属性,这是一个预定义好的、类型为 Symbol 的特殊值,所以要放在方括号内。

  • 手动部署
    在部署iterator接口的时候，在[Symbol.iterator]函数中返回的对象要有next方法，在这个next方法中，又可以返回{value: xx, done: xx}，例子可见《ES6标准入门》P160

  • 调用Iterator接口的场合

    • 解构赋值

    • 扩展运算符
      只要某个数据结构部署了 Iterator 接口,就可以对它使用扩展运算符,将其转为数组。

    • yield*
      yield* 后面跟的是一个可遍历的结构,它会调用该结构的遍历器接口。

    • 数组的遍历回调用遍历器接口，对任何接受数组作为参数的场合，都调用了遍历器接口。

      • for...of
      • Arry.from
      • Map(), Set(), WeakMap(), WeakSet() (比如new Map([[‘a’,1],[‘b’,2]]) )
      • Promise.all()
      • Promise.race()

字符串的Iterator接口

字符串是一个类似数组的对象，原生具有iterator接口。

对于原生的iterator接口可以进行覆盖，从而修改遍历器的行为。修改后的对于使用到遍历器的场合，原来的行为都会被修改，比如扩展运算符等。

iterator接口和Generator函数

对Symbol.iterator方法最简单的实现是Generator函数。

遍历器对象的return()和throw()

在遍历器对象生成函数中，next()是必须部署的，return()和throw()是可选的

  • return()：
    for...of循环提前退出时(出错，有break，或continue)，就会进行调用，如果一个对象在完成遍历前需要清理和释放资源，可以部署return()方法。
  • throw()：
    配合Generator函数使用，一般遍历器不使用

for...of循环

for...of 循环内部调用的是数据结构的Symbol.iterator方法，有这个方法就可使用for...of

  • 数组：for...of本质就是调用这个接口产生的遍历器

for…in 循环读取键名,for…of 循环读取键值。如果要通过for…of 循环,获取数组的索引,可以借助数组实例的entries 方法和keys 方法。

  • 数组，Set，Map都部署的三个方法
    entries()，keys()，values()都是返回遍历器对象

  • 可以使用for...of的 类数组对象(具备iterator接口的类数组对象)

    • 字符串
      可以正确识别32位UTF-16字符

    • DOM NodeList

    • arguments
      不具备iterator接口的类数组对象的解决方法：
      使用Arry.from将其转为数组

  • 对象
    普通对象可以用for...in遍历键名，不能用for...of
    解决方法：

    • Object.keys将键名生成一个数组，遍历该数组

    • 部署iterator接口
      其中一个方便的方法是把数组的Symbol.iterator属性赋值给其他对象的Symbol.iterator属性

    • 使用Generator函数对对象进行包装

遍历语法的比较

  • for循环

  • 数组内置的forEach，但无法在中途跳出循环

  • for...in 遍历数组键名
    for...in是为遍历对象设计，不适用于数组。缺点：

    • 数组的键名是数组，但是for...in循环是以字符串作为键名“0”,“1”…
    • for...in 循环不仅遍历数字键名,还会遍历手动添加的其他键,甚至包括原型链上的键。
    • 某些情况下, for…in 循环会以任意顺序遍历键名。

  • for...in优点

    • 有着同 for…in 一样的简洁语法,但是没有 for…in 那些缺点。
    • 不同用于 forEach 方法,它可以与 break 、 continue 和 return 配合使用。
    • 提供了遍历所有数据结构的统一操作接口。

box-shadow

作用：
  • 营造立体感
  • 充当没有宽度的边框，不会影响布局。
  • 特殊效果（多投影）
    浏览器实现阴影效果原理：

上图从左至右依次是浏览器的4步动作。首先根据指定的color克隆一个和原始元素同样尺寸的元素。然后根据指定的x-offset / y-offset将克隆出来的元素进行偏移。
第三步根据指定的blur-radius模糊半径10px，依据高斯算法进行模糊处理。本质上是在阴影边缘将阴影色和纯透明色之间的颜色过渡，长度近似于blur-radius模糊半径的两倍，即20px左右。这就是导致阴影外扩的原因。
最后，浏览器在克隆元素上，将交集部分剪切掉，剩余部分拼接到原始元素边上，就实现了最终的效果。注意这和我们看上去的有的不同，看上去像是两个元素叠加在了一次，导致下层的克隆元素部分被遮盖了。其实不是的。这意味着即使你将原始元素的背景色设成半透明，也不会看到底层的克隆元素，因为原始元素的下面根本没有任何东西。

参数：
  • inset： 内阴影
  • x-offset：水平偏移量
  • y-offset：垂直偏移量
  • blur-radius：阴影模糊半径
  • spread-radius：阴影拓展半径，即是使阴影放大或缩小
    阴影模糊会导致阴影外扩，此时就可以使用spread-radius缩小阴影大小，使不需要的外扩隐藏

图片加阴影：

发表于 2018-01-17

float布局：

  • 元素“浮动”
  • 脱离文档流
  • 但不脱离文件流
    脱离文档流是指“很长文字”部分所占空间是外部div是所有部分，float 部分不占空间，不脱离文件流是指float的元素把“很长文字”部分的文字挤出。
对自身的影响：
  • 形成“块”(BFC)
    形成块以后负责自身的布局，可以设置宽高，即使一个元素是inline的，使用float后，也可以设置宽高。
  • 位置尽量靠上
  • 位置尽量靠左（右）
对兄弟的影响：
  • 上面贴非float元素
  • 旁边贴float元素
  • 不影响其他块级元素位置
  • 影响其他块级元素内部文本

假如某个div元素A是浮动的，如果A元素上一个元素也是浮动的，那么A元素会跟随在上一个元素的后边(如果一行放不下这两个元素，那么A元素会被挤到下一行)；如果A元素上一个元素是标准流中的元素，那么A的相对垂直位置不会改变，也就是说A的顶部总是和上一个元素的底部对齐。
div的顺序是HTML代码中div的顺序决定的。

这也是在float三栏布局中在代码中中间部分是放在最后，右边部分是放在中间部分前面的原因。
