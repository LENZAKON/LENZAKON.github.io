布局（二）float布局

发表于 2018-01-17

float布局：

  • 元素“浮动”
  • 脱离文档流
  • 但不脱离文件流
    脱离文档流是指“很长文字”部分所占空间是外部div是所有部分，float 部分不占空间，不脱离文件流是指float的元素把“很长文字”部分的文字挤出。
对自身的影响：
  • 形成“块”(BFC)
    形成块以后负责自身的布局，可以设置宽高，即使一个元素是inline的，使用float后，也可以设置宽高。
  • 位置尽量靠上
  • 位置尽量靠左（右）
对兄弟的影响：
  • 上面贴非float元素
  • 旁边贴float元素
  • 不影响其他块级元素位置
  • 影响其他块级元素内部文本

假如某个div元素A是浮动的，如果A元素上一个元素也是浮动的，那么A元素会跟随在上一个元素的后边(如果一行放不下这两个元素，那么A元素会被挤到下一行)；如果A元素上一个元素是标准流中的元素，那么A的相对垂直位置不会改变，也就是说A的顶部总是和上一个元素的底部对齐。
div的顺序是HTML代码中div的顺序决定的。

这也是在float三栏布局中在代码中中间部分是放在最后，右边部分是放在中间部分前面的原因。

对父级元素的影响：
  • 从布局上“消失”

  • 高度塌陷

    • overflow: auto / hidden

    • 伪元素：
      原理：让父级元素中有一个内容，刚好把父级元素撑开

      对于CSS的清除浮动(clear)，一定要牢记：这个规则只能影响使用清除的元素本身，不能影响其他元素。

非布局样式——行高

发表于 2018-01-16

行高：

行高是指一行的高度，这一行的高度是由行内元素中line-height最高的元素决定的，在一行中，不同的line-height的元素，渲染的高度是一样的，底线和顶线之间，字是根据基线对准的。

  • 当行高大于字体的时候，行高是分布在字的上下两部分，字为垂直居中

  • 不同的inline-box的字之间默认是根据基线对齐的。可以通过vertical-align来设置对齐方式。
    注意：在设置的时候要对不同的inline-box同时进行设置，否则可能达不到预期效果。vertical-align默认为base-line，可以设置：top，bottom，middle，也可以是px，但不推荐，top不一定是文字的最上部，是按顶线对齐的。bottom是底线。

  • 图片底部3px空白问题：

    由于img是一个inline元素，按基线对齐，基线和底线之间的距离就是这个空白，这个距离视字体大小而定，如果字体为12px，则空白大概为3px。
    解决：

    • img的vertical-aline设为bottom
    • img的display设为block，不会再按字体一样对齐。

HTML

发表于 2018-01-13

元素嵌套问题：

  • 块级元素可以包含行内元素

  • 块级元素不一定能包含块级元素，如p中不能包含div

  • 行内元素一般不能包含块级元素
    例外的是a标签：在a中可以包含块级元素
    查看方式：
    在w3c官方文档中可以查看：

    • 分类 Categories
    • 内容模型（决定嵌套关系） Content model

    比如a的Content model为Transparent，表示a在计算嵌套合法性的时候a是不存在的，所以在<a><div></div></a>中，由于忽略了a，所以是合法的。

    再比如在<p><div></div></p>中p的Content model为Transparent为Phrasing content.其中不包含div，故为不合法的。

元素分类：

  • 按默认样式分类：

    • block

    • inline(文本元素)
      inline的形状有时是不规则的，根据空间情况而定，当inline元素空间不够时会分行，其x形状则是上面部分长方形和下面部分长方形。不可以设置宽高

    • inline-block (表单元素)
      表单标签select等，对外表现和inline元素一样，对内是一个规则方形的元素，宽度不够时也不会把元素变成两半，一半在上一半在下，还也可设置宽高

  • 按内容分类：
    w3c分类

doctype意义：

  • 让浏览器以标准模式进行渲染
  • 让浏览器知道元素的合法性

HTML XHTML HTML5的关系：

  • HTML属于SGML
  • XHTML属于XML，是HTML进行XML严格化的结果
  • HTML5既不属于SGML，也不属于XML，比XHTML宽松

HTML5有什么变化：

  • 新的语义化元素
  • 表单增强
  • 新的API（离线，音视频，图形，实时通信，本地存储，设备能力）
  • 分类和嵌套变更

em和i的区别：

  • em是语义化的标签，表强调
  • i是纯样式标签，表斜体
  • HTML5中i不推荐使用，一般用作图标

语义化的意义：

  • 开发者容易理解
  • 机器容易理解（搜索，读屏软件）
  • 有助于SEO
  • semantic microdata （在HTML中添加标注，标注这个东西是什么，对元素进行进一步的语义化）

哪些元素可以自闭合：

  • 表单元素input
  • 图片img
  • br换行 hr水平线
  • meta link

HTML和DOM的关系：

  • HTML是“死”的
  • DOM是由HTML解析而来，是活的
  • JS可以维护DOM

property和attribute的区别：

在HTML中，标签的属性为attribute，经过DOM的解析，变成property

  • attribute是“死”的
  • property是活的
  • 两者不会互相影响，在开发者工具中设置其中一个值，不会影响另一个的值

form的作用：

  • 直接提交表单
  • 使用submit/reset按钮
  • 便于浏览器保存表单
  • 第三方库可以整体提取值
  • 第三方库可以进行表单验证
